# Rust 教程 - 变量详解

## 本章概述

变量是编程语言中最基础的概念之一。Rust 的变量系统有一些独特的特性，比如默认不可变性和变量遮蔽。本章将深入讲解 Rust 中变量的各种用法和特性。

## 学习目标

- 理解 Rust 变量的不可变性
- 掌握可变变量的使用
- 理解变量遮蔽的概念
- 学会使用常量
- 掌握类型标注
- 理解变量作用域
- 掌握解构赋值

## 1. 变量声明

在 Rust 中，使用 `let` 关键字声明变量：

```rust
let x = 5;
println!("x = {}", x);
```

**重要特性**：Rust 变量默认是不可变的（immutable）。这是 Rust 的一个核心设计理念，旨在提供更安全的并发编程。

```rust
let x = 5;
x = 6;  // 错误！不能对不可变变量重新赋值
```

## 2. 可变变量

如果需要修改变量的值，需要使用 `mut` 关键字：

```rust
let mut y = 5;
println!("y 初始值: {}", y);

y = 10;  // 可变变量可以重新赋值
println!("y 修改后: {}", y);
```

**使用场景**：
- 需要在循环中累加值
- 需要修改配置参数
- 需要更新状态

## 3. 变量遮蔽（Variable Shadowing）

Rust 允许声明一个与之前变量同名的新变量，这称为变量遮蔽：

```rust
let spaces = "   ";       // 字符串类型
println!("spaces 是字符串: '{}'", spaces);

let spaces = spaces.len(); // 新变量遮蔽了旧变量，类型可以改变！
println!("spaces 现在是数字: {}", spaces);
```

**遮蔽 vs mut 的区别**：

| 特性 | 遮蔽 | mut |
|------|------|-----|
| 创建新变量 | 是 | 否 |
| 可以改变类型 | 是 | 否 |
| 内存位置 | 可能不同 | 相同 |

**示例对比**：

```rust
// 使用遮蔽 - 可以改变类型
let spaces = "   ";
let spaces = spaces.len();  // 合法

// 使用 mut - 不能改变类型
let mut spaces = "   ";
spaces = spaces.len();  // 错误！类型不匹配
```

## 4. 常量

常量使用 `const` 关键字声明，必须注明类型：

```rust
const MAX_POINTS: u32 = 100_000;
println!("最大分数: {}", MAX_POINTS);
```

**常量 vs 变量**：

| 特性 | 常量 | 变量 |
|------|------|------|
| 可变性 | 永远不可变 | 默认不可变，可用 mut |
| 类型标注 | 必须 | 可选（编译器推断） |
| 作用域 | 可在任何作用域 | 受限于定义的作用域 |
| 赋值时机 | 编译时确定 | 运行时确定 |
| 命名约定 | SCREAMING_SNAKE_CASE | snake_case |

**常量的优势**：
- 编译时已知值，性能更好
- 全局可见，便于共享
- 意图明确，不可修改

## 5. 类型标注

虽然 Rust 有强大的类型推断系统，但有时需要显式标注类型：

```rust
let z: i32 = 10;           // 显式标注整数类型
let pi: f64 = 3.14159;     // 显式标注浮点类型
let is_rust_fun: bool = true;  // 显式标注布尔类型
let letter: char = '🦀';    // 显式标注字符类型
```

**何时需要类型标注**：
- 编译器无法推断类型时
- 需要特定类型时（如 `f32` 而非 `f64`）
- 提高代码可读性

## 6. 变量作用域

变量的作用域是指变量有效的范围：

```rust
let outer_var = "外部变量";

{
    let inner_var = "内部变量";
    println!("内部作用域可以访问: {}", outer_var);
    println!("内部作用域可以访问: {}", inner_var);
}

println!("外部作用域可以访问: {}", outer_var);
// println!("{}", inner_var);  // 错误！内部变量已离开作用域
```

**作用域规则**：
- 变量从声明处开始有效
- 变量在所属作用域结束时失效
- 内部作用域可以访问外部变量
- 外部作用域不能访问内部变量

## 7. 未使用的变量

Rust 编译器会警告未使用的变量。可以使用下划线前缀消除警告：

```rust
let _unused_var = 42;  // 不会产生警告
let _ = "这个值会被丢弃";  // 下划线本身会丢弃任何值
```

**实际应用**：
- 在开发过程中临时注释代码
- 接收但不使用函数返回值
- 在模式匹配中忽略某些值

## 8. 解构赋值

Rust 支持从元组和数组中解构值：

**元组解构**：
```rust
let (a, b, c) = (1, 2, 3);
println!("a = {}, b = {}, c = {}", a, b, c);
```

**数组解构**：
```rust
let array = [10, 20, 30];
let [first, ..] = array;
println!("数组第一个元素: {}", first);
```

**复杂解构**：
```rust
let (x, (y, z)) = (1, (2, 3));
println!("x = {}, y = {}, z = {}", x, y, z);
```

## 9. 变量命名规则

Rust 变量命名必须遵循以下规则：

**合法命名**：
- 必须以字母或下划线开头
- 可以包含字母、数字、下划线
- 推荐 snake_case 命名风格

```rust
let snake_case = "蛇形命名（推荐）";
let _starts_with_underscore = "下划线开头";
let _has_number_123 = "可以包含数字";
```

**非法命名**：
```rust
// let 1starts_with_number = 1;  // 错误！不能以数字开头
// let has-dash = 1;              // 错误！不能包含连字符
// let let = 1;                   // 错误！不能使用关键字
```

## 10. 最佳实践

### 优先使用不可变变量
```rust
// 好 - 默认不可变
let count = 10;

// 避免 - 除非真的需要修改
let mut count = 10;
```

### 合理使用遮蔽
```rust
// 好 - 类型转换
let data = "42";
let data: i32 = data.parse().unwrap();

// 避免 - 容易混淆
let x = 1;
let x = 2;
let x = 3;
```

### 使用有意义的变量名
```rust
// 好
let user_count = 100;
let max_retries = 3;

// 避免
let n = 100;
let x = 3;
```

## 常见问题

### Q1: 为什么 Rust 默认变量不可变？

**答**：这是 Rust 的设计哲学，为了：
- 减少运行时错误
- 提高并发安全性
- 优化编译器优化

### Q2: 遮蔽和重新赋值有什么区别？

**答**：
- 遮蔽创建新变量，可以改变类型
- 重新赋值修改现有变量的值，不能改变类型

### Q3: 什么时候使用常量，什么时候使用不可变变量？

**答**：
- 使用常量：全局共享、编译时已知、语义上不可变
- 使用不可变变量：局部作用域、运行时计算、可能需要遮蔽
