# Cargo 教程 - Rust 包管理器

Cargo 是 Rust 的官方包管理器和构建系统，是每个 Rust 开发者必须掌握的工具。

## Cargo 简介

Cargo 主要功能：
- **项目管理**：创建、组织 Rust 项目
- **依赖管理**：下载和管理外部库（crates）
- **构建系统**：编译代码、运行测试
- **发布工具**：发布库到 crates.io

## 创建项目

### 创建二进制项目

```bash
# 创建可执行项目
cargo new my_project

# 项目结构
my_project/
├── Cargo.toml      # 项目配置文件
├── src/
│   └── main.rs     # 主程序入口
└── .gitignore      # Git 忽略文件
```

### 创建库项目

```bash
# 创建库项目（不可执行）
cargo new --lib my_lib

# 项目结构
my_lib/
├── Cargo.toml
├── src/
│   └── lib.rs      # 库入口文件
└── .gitignore
```

### 在现有目录创建项目

```bash
# 在当前目录初始化项目
cargo init

# 指定项目名称
cargo init my_project
```

## Cargo.toml 配置文件

`Cargo.toml` 是项目的配置文件，使用 TOML 格式。

### 基本结构

```toml
[package]
name = "my_project"        # 项目名称
version = "0.1.0"          # 版本号（遵循语义化版本）
edition = "2021"           # Rust 版本（2015, 2018, 2021）
authors = ["Your Name <you@example.com>"]
description = "A sample project"
license = "MIT"
repository = "https://github.com/user/my_project"
readme = "README.md"
keywords = ["rust", "example"]
categories = ["command-line-utilities"]

[dependencies]
# 项目依赖

[dev-dependencies]
# 开发依赖（仅用于测试和示例）

[build-dependencies]
# 构建脚本依赖

[features]
# 条件编译特性

[profile.release]
# 发布配置
```

### 依赖管理

#### 添加依赖

**方式一：手动编辑 Cargo.toml**

```toml
[dependencies]
serde = "1.0"                    # 指定版本
rand = "0.8.5"                   # 指定精确版本
tokio = { version = "1.0", features = ["full"] }  # 启用特性
serde_json = { git = "https://github.com/serde-rs/json" }  # Git 仓库
my_local_crate = { path = "../my_local_crate" }   # 本地路径
```

**方式二：使用命令行**

```bash
# 添加依赖
cargo add serde

# 添加指定版本
cargo add serde@1.0

# 添加并启用特性
cargo add tokio --features full

# 添加开发依赖
cargo add --dev proptest

# 添加构建依赖
cargo add --build cc
```

#### 版本指定规则

Cargo 使用语义化版本（SemVer）：

```toml
[dependencies]
serde = "1.2.3"     # 精确版本：1.2.3
serde = "1.2"       # 范围：>=1.2.0, <2.0.0
serde = "1"         # 范围：>=1.0.0, <2.0.0
serde = ">=1.2.3"   # 最小版本：>=1.2.3
serde = "<1.3"      # 最大版本：<1.3.0
serde = "~1.2.3"    # 补丁版本兼容：>=1.2.3, <1.3.0
serde = "^1.2.3"    # 与 serde = "1.2.3" 相同
serde = "*"         # 任意版本（不推荐）
```

#### 查看依赖

```bash
# 查看依赖树
cargo tree

# 查看过时的依赖
cargo outdated  # 需要安装：cargo install cargo-outdated
```

#### 更新依赖

```bash
# 更新所有依赖到最新兼容版本
cargo update

# 更新特定依赖
cargo update -p serde

# 锁定依赖版本
# Cargo.lock 文件会锁定依赖版本，通常不需要手动编辑
```

### 特性（Features）

特性用于条件编译和可选依赖：

```toml
[features]
default = ["json"]           # 默认启用的特性
json = ["serde_json"]        # json 特性需要 serde_json
full = ["json", "yaml"]      # full 特性包含多个特性
yaml = ["serde_yaml"]

[dependencies]
serde = { version = "1.0", optional = true }  # 可选依赖
serde_json = { version = "1.0", optional = true }
serde_yaml = { version = "0.8", optional = true }
```

**使用特性**：

```bash
# 使用默认特性
cargo build

# 禁用默认特性
cargo build --no-default-features

# 启用特定特性
cargo build --features json

# 启用所有特性
cargo build --all-features
```

**代码中使用特性**：

```rust
#[cfg(feature = "json")]
fn use_json() {
    // json 特性相关的代码
}

#[cfg(feature = "yaml")]
fn use_yaml() {
    // yaml 特性相关的代码
}
```

## 构建项目

### 编译代码

```bash
# 开发模式编译（快速编译，优化较少）
cargo build

# 发布模式编译（慢速编译，高度优化）
cargo build --release

# 检查代码（不生成二进制，速度最快）
cargo check

# 查看编译详细信息
cargo build --verbose
```

### 运行项目

```bash
# 编译并运行
cargo run

# 运行发布版本
cargo run --release

# 传递参数给程序
cargo run -- arg1 arg2

# 运行特定 binary
cargo run --bin my_binary
```

### 清理构建

```bash
# 删除 target 目录
cargo clean
```

## 项目结构

### 多 binary 项目

```
my_project/
├── Cargo.toml
├── src/
│   ├── main.rs       # 默认 binary
│   ├── bin/
│   │   ├── extra.rs  # 额外的 binary
│   │   └── tool.rs   # 另一个 binary
│   └── lib.rs        # 库代码
```

**Cargo.toml**:

```toml
[[bin]]
name = "my_project"
path = "src/main.rs"

[[bin]]
name = "extra"
path = "src/bin/extra.rs"

[[bin]]
name = "tool"
path = "src/bin/tool.rs"
```

**运行不同的 binary**:

```bash
cargo run --bin my_project
cargo run --bin extra
cargo run --bin tool
```

### 工作空间（Workspace）

工作空间用于管理多个相关的包：

```toml
# 根目录 Cargo.toml
[workspace]
members = [
    "app",
    "lib1",
    "lib2",
]
```

**目录结构**:

```
workspace/
├── Cargo.toml      # 工作空间配置
├── Cargo.lock      # 共享的锁文件
├── app/
│   ├── Cargo.toml
│   └── src/
│       └── main.rs
├── lib1/
│   ├── Cargo.toml
│   └── src/
│       └── lib.rs
└── lib2/
    ├── Cargo.toml
    └── src/
        └── lib.rs
```

**工作空间命令**:

```bash
# 在根目录运行命令会应用到所有成员
cargo build

# 只构建特定包
cargo build -p lib1

# 运行特定包
cargo run -p app
```

## 测试

### 运行测试

```bash
# 运行所有测试
cargo test

# 运行特定测试
cargo test test_name

# 运行特定模块的测试
cargo test module::

# 显示 println! 输出
cargo test -- --nocapture

# 并行测试
cargo test -- --test-threads=4

# 运行基准测试
cargo test --benches
```

### 测试组织

```rust
// 单元测试 - 在源文件中
#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_function() {
        assert_eq!(2 + 2, 4);
    }
}

// 集成测试 - 在 tests/ 目录
// tests/integration_test.rs
use my_lib::some_function;

#[test]
fn test_integration() {
    assert!(some_function());
}
```

## 文档

### 生成文档

```bash
# 生成文档
cargo doc

# 生成并打开文档
cargo doc --open

# 包含依赖的文档
cargo doc --document-private-items
```

### 编写文档注释

```rust
/// 这是一个文档注释
/// 
/// # Examples
/// 
/// ```
/// let result = add(2, 3);
/// assert_eq!(result, 5);
/// ```
fn add(a: i32, b: i32) -> i32 {
    a + b
}

/// 这是一个模块级别的文档注释
//! 
//! # 模块功能
//! 
//! 这个模块提供了数学计算功能。

/// 结构体文档
/// 
/// 描述结构体的用途
pub struct MyStruct {
    /// 字段说明
    pub field: i32,
}

impl MyStruct {
    /// 方法说明
    pub fn new() -> Self {
        Self { field: 0 }
    }
}
```

## Cargo 配置

### 全局配置

在 `~/.cargo/config` 或 `.cargo/config.toml`：

```toml
[build]
jobs = 4                      # 并行编译数
rustflags = ["-A", "warnings"] # 编译器标志

[net]
retry = 5                     # 网络重试次数
git-fetch-with-cli = true     # 使用系统 git

[http]
proxy = "127.0.0.1:7890"      # HTTP 代理
check-revoke = false          # 不检查证书撤销

[term]
verbose = false               # 详细输出
color = 'auto'                # 彩色输出

[source.crates-io]
replace-with = 'ustc'         # 使用镜像源

[source.ustc]
registry = "https://mirrors.ustc.edu.cn/crates.io-index"
```

### 项目配置

在项目根目录 `.cargo/config.toml`：

```toml
[build]
target = "x86_64-unknown-linux-gnu"

[target.x86_64-pc-windows-msvc]
linker = "rust-lld"           # 使用更快的链接器

[alias]
b = "build"
c = "check"
t = "test"
r = "run"
rr = "run --release"
```

## 最佳实践

### 1. 使用语义化版本

```toml
[dependencies]
serde = "1.0"                 # 主版本号，允许小版本更新
serde = "~1.0.100"            # 补丁版本兼容
serde = ">=1.0.100, <2.0.0"   # 明确范围
```

### 2. 最小化依赖

- 只添加真正需要的依赖
- 使用特性（features）控制可选依赖
- 定期检查并移除未使用的依赖

### 3. 使用 cargo-watch 提高开发效率

```bash
# 安装
cargo install cargo-watch

# 自动编译检查
cargo watch -x check

# 自动运行
cargo watch -x run

# 自动测试
cargo watch -x test
```

### 4. 保持 Cargo.lock

- **库项目**：通常不提交 `Cargo.lock`（用户会生成自己的）
- **二进制项目**：应该提交 `Cargo.lock`（确保可重现构建）

### 5. 使用工作空间管理大型项目

```toml
[workspace]
members = ["app", "core", "utils"]

[workspace.dependencies]
serde = { version = "1.0", features = ["derive"] }
tokio = { version = "1.0", features = ["full"] }
```

## 常见问题

### 1. 编译缓慢

**解决方案**：
- 使用 `cargo check` 而不是 `cargo build`
- 在 Windows 上使用更快的链接器
- 减少并行编译数（如果内存不足）：`CARGO_BUILD_JOBS=2 cargo build`

### 2. 依赖冲突

**解决方案**：
```bash
# 查看依赖树
cargo tree

# 查看重复依赖
cargo tree -d

# 强制使用统一版本
cargo update -p conflicting-crate
```

### 3. 网络问题

**解决方案**：
- 配置镜像源（见上文）
- 使用代理：`export HTTP_PROXY=127.0.0.1:7890`
- 使用离线模式：`cargo build --offline`
