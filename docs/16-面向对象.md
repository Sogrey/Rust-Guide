# Rust 面向对象

Rust 不是传统的面向对象语言，但它支持面向对象编程的许多概念，并通过独特的方式实现封装、继承和多态。

## 面向对象概念

### 什么是面向对象？

面向对象编程（OOP）通常包含以下特性：

1. **封装（Encapsulation）**：隐藏实现细节，暴露公共接口
2. **继承（Inheritance）**：代码复用和建立 is-a 关系
3. **多态（Polymorphism）**：同一接口不同实现
4. **抽象（Abstraction）**：简化复杂系统

### Rust 的面向对象特性

- ✅ **封装**：通过 `pub` 关键字控制可见性
- ⚠️ **继承**：Rust 没有类继承，使用 trait 和组合
- ✅ **多态**：通过 trait 和泛型实现
- ✅ **抽象**：通过 trait 和关联类型

## 封装

### 结构体封装

```rust
// 定义一个封装的用户类型
pub struct User {
    username: String,        // 私有字段
    email: String,           // 私有字段
    active: bool,            // 私有字段
    sign_in_count: u64,      // 私有字段
}

impl User {
    // 公共构造函数
    pub fn new(username: String, email: String) -> Self {
        User {
            username,
            email,
            active: true,
            sign_in_count: 0,
        }
    }
    
    // 公共方法 - getter
    pub fn username(&self) -> &str {
        &self.username
    }
    
    pub fn email(&self) -> &str {
        &self.email
    }
    
    pub fn is_active(&self) -> bool {
        self.active
    }
    
    // 公共方法 - setter（可变借用）
    pub fn set_email(&mut self, email: String) {
        self.email = email;
    }
    
    // 公共方法 - 业务逻辑
    pub fn sign_in(&mut self) {
        self.sign_in_count += 1;
        self.active = true;
    }
    
    pub fn sign_out(&mut self) {
        self.active = false;
    }
    
    // 私有方法
    fn validate_email(&self) -> bool {
        self.email.contains('@')
    }
}

fn main() {
    let mut user = User::new("alice".to_string(), "alice@example.com".to_string());
    
    // 使用公共接口
    println!("Username: {}", user.username());
    println!("Email: {}", user.email());
    
    user.sign_in();
    println!("Active: {}", user.is_active());
    
    user.set_email("newemail@example.com".to_string());
    
    // 无法访问私有字段
    // user.username = "bob".to_string();  // 错误！
    // println!("{}", user.sign_in_count);  // 错误！
}
```

## 继承模拟

Rust 没有类继承，但可以通过以下方式实现类似效果：

### 1. Trait 继承

```rust
// 基础 trait
trait Animal {
    fn name(&self) -> &str;
    fn speak(&self);
}

// 派生 trait
trait Pet: Animal {
    fn owner(&self) -> &str;
    fn play(&self);
}

// 实现基础 trait
struct Dog {
    name: String,
    owner: String,
}

impl Animal for Dog {
    fn name(&self) -> &str {
        &self.name
    }
    
    fn speak(&self) {
        println!("{} says: Woof!", self.name);
    }
}

// 实现 Pet trait（必须先实现 Animal）
impl Pet for Dog {
    fn owner(&self) -> &str {
        &self.owner
    }
    
    fn play(&self) {
        println!("{} plays with {}", self.name, self.owner);
    }
}

fn main() {
    let dog = Dog {
        name: "Buddy".to_string(),
        owner: "Alice".to_string(),
    };
    
    dog.speak();
    dog.play();
}
```

### 2. 组合优于继承

```rust
// 组合模式
struct Engine {
    horsepower: u32,
}

impl Engine {
    fn start(&self) {
        println!("Engine started ({} hp)", self.horsepower);
    }
}

struct Wheels {
    count: u32,
}

impl Wheels {
    fn roll(&self) {
        println!("Rolling on {} wheels", self.count);
    }
}

// 组合多个组件
struct Car {
    engine: Engine,
    wheels: Wheels,
    model: String,
}

impl Car {
    fn new(model: String, horsepower: u32) -> Self {
        Car {
            engine: Engine { horsepower },
            wheels: Wheels { count: 4 },
            model,
        }
    }
    
    fn drive(&self) {
        println!("Driving {}", self.model);
        self.engine.start();
        self.wheels.roll();
    }
}

fn main() {
    let car = Car::new("Tesla Model 3".to_string(), 300);
    car.drive();
}
```

### 3. 使用 trait 默认实现

```rust
trait Drawable {
    fn draw(&self);
    
    // 默认实现
    fn describe(&self) {
        println!("This is a drawable object");
    }
    
    fn area(&self) -> f64 {
        0.0
    }
}

struct Circle {
    radius: f64,
}

impl Drawable for Circle {
    fn draw(&self) {
        println!("Drawing circle with radius {}", self.radius);
    }
    
    // 重写默认实现
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }
}

struct Square {
    side: f64,
}

impl Drawable for Square {
    fn draw(&self) {
        println!("Drawing square with side {}", self.side);
    }
    
    fn area(&self) -> f64 {
        self.side * self.side
    }
}

fn main() {
    let circle = Circle { radius: 5.0 };
    let square = Square { side: 3.0 };
    
    circle.draw();
    circle.describe();
    println!("Circle area: {}", circle.area());
    
    square.draw();
    square.describe();
    println!("Square area: {}", square.area());
}
```

## 多态

### 通过 Trait 对象实现多态

```rust
trait Shape {
    fn area(&self) -> f64;
    fn perimeter(&self) -> f64;
}

struct Rectangle {
    width: f64,
    height: f64,
}

impl Shape for Rectangle {
    fn area(&self) -> f64 {
        self.width * self.height
    }
    
    fn perimeter(&self) -> f64 {
        2.0 * (self.width + self.height)
    }
}

struct Circle {
    radius: f64,
}

impl Shape for Circle {
    fn area(&self) -> f64 {
        std::f64::consts::PI * self.radius * self.radius
    }
    
    fn perimeter(&self) -> f64 {
        2.0 * std::f64::consts::PI * self.radius
    }
}

// 使用 trait 对象实现多态
fn print_shape_info(shape: &dyn Shape) {
    println!("Area: {}, Perimeter: {}", shape.area(), shape.perimeter());
}

// 使用泛型（静态分发）
fn print_shape_info_generic<T: Shape>(shape: &T) {
    println!("Area: {}, Perimeter: {}", shape.area(), shape.perimeter());
}

fn main() {
    let rectangle = Rectangle { width: 5.0, height: 3.0 };
    let circle = Circle { radius: 2.0 };
    
    // 动态分发
    print_shape_info(&rectangle);
    print_shape_info(&circle);
    
    // 静态分发
    print_shape_info_generic(&rectangle);
    print_shape_info_generic(&circle);
    
    // 使用集合存储不同类型
    let shapes: Vec<Box<dyn Shape>> = vec![
        Box::new(Rectangle { width: 5.0, height: 3.0 }),
        Box::new(Circle { radius: 2.0 }),
    ];
    
    for shape in &shapes {
        println!("Area: {}", shape.area());
    }
}
```

### 使用泛型实现多态

```rust
trait Processor {
    fn process(&self, data: &str) -> String;
}

struct UpperCaseProcessor;
struct LowerCaseProcessor;
struct ReverseProcessor;

impl Processor for UpperCaseProcessor {
    fn process(&self, data: &str) -> String {
        data.to_uppercase()
    }
}

impl Processor for LowerCaseProcessor {
    fn process(&self, data: &str) -> String {
        data.to_lowercase()
    }
}

impl Processor for ReverseProcessor {
    fn process(&self, data: &str) -> String {
        data.chars().rev().collect()
    }
}

// 泛型函数 - 静态分发
fn process_data<P: Processor>(processor: &P, data: &str) -> String {
    processor.process(data)
}

// 动态分发版本
fn process_data_dyn(processor: &dyn Processor, data: &str) -> String {
    processor.process(data)
}

fn main() {
    let data = "Hello, Rust!";
    
    let upper = UpperCaseProcessor;
    let lower = LowerCaseProcessor;
    let reverse = ReverseProcessor;
    
    println!("Upper: {}", process_data(&upper, data));
    println!("Lower: {}", process_data(&lower, data));
    println!("Reverse: {}", process_data(&reverse, data));
    
    // 运行时选择
    let processors: Vec<Box<dyn Processor>> = vec![
        Box::new(UpperCaseProcessor),
        Box::new(LowerCaseProcessor),
        Box::new(ReverseProcessor),
    ];
    
    for processor in &processors {
        println!("Result: {}", processor.process(data));
    }
}
```

## 设计模式

### 工厂模式

```rust
// 产品 trait
trait Product {
    fn operation(&self) -> String;
}

// 具体产品
struct ConcreteProductA;
struct ConcreteProductB;

impl Product for ConcreteProductA {
    fn operation(&self) -> String {
        "Product A".to_string()
    }
}

impl Product for ConcreteProductB {
    fn operation(&self) -> String {
        "Product B".to_string()
    }
}

// 工厂
struct Factory;

impl Factory {
    fn create_product(product_type: &str) -> Box<dyn Product> {
        match product_type {
            "A" => Box::new(ConcreteProductA),
            "B" => Box::new(ConcreteProductB),
            _ => panic!("Unknown product type"),
        }
    }
}

fn main() {
    let product_a = Factory::create_product("A");
    let product_b = Factory::create_product("B");
    
    println!("{}", product_a.operation());
    println!("{}", product_b.operation());
}
```

### 策略模式

```rust
// 策略 trait
trait Strategy {
    fn execute(&self, a: i32, b: i32) -> i32;
}

// 具体策略
struct AddStrategy;
struct SubtractStrategy;
struct MultiplyStrategy;

impl Strategy for AddStrategy {
    fn execute(&self, a: i32, b: i32) -> i32 {
        a + b
    }
}

impl Strategy for SubtractStrategy {
    fn execute(&self, a: i32, b: i32) -> i32 {
        a - b
    }
}

impl Strategy for MultiplyStrategy {
    fn execute(&self, a: i32, b: i32) -> i32 {
        a * b
    }
}

// 上下文
struct Context {
    strategy: Box<dyn Strategy>,
}

impl Context {
    fn new(strategy: Box<dyn Strategy>) -> Self {
        Context { strategy }
    }
    
    fn set_strategy(&mut self, strategy: Box<dyn Strategy>) {
        self.strategy = strategy;
    }
    
    fn execute_strategy(&self, a: i32, b: i32) -> i32 {
        self.strategy.execute(a, b)
    }
}

fn main() {
    let mut context = Context::new(Box::new(AddStrategy));
    println!("10 + 5 = {}", context.execute_strategy(10, 5));
    
    context.set_strategy(Box::new(SubtractStrategy));
    println!("10 - 5 = {}", context.execute_strategy(10, 5));
    
    context.set_strategy(Box::new(MultiplyStrategy));
    println!("10 * 5 = {}", context.execute_strategy(10, 5));
}
```

### 观察者模式

```rust
// 观察者 trait
trait Observer {
    fn update(&self, message: &str);
}

// 具体观察者
struct EmailObserver {
    email: String,
}

impl Observer for EmailObserver {
    fn update(&self, message: &str) {
        println!("Sending email to {}: {}", self.email, message);
    }
}

struct SmsObserver {
    phone: String,
}

impl Observer for SmsObserver {
    fn update(&self, message: &str) {
        println!("Sending SMS to {}: {}", self.phone, message);
    }
}

// 主题
struct Subject {
    observers: Vec<Box<dyn Observer>>,
}

impl Subject {
    fn new() -> Self {
        Subject { observers: Vec::new() }
    }
    
    fn attach(&mut self, observer: Box<dyn Observer>) {
        self.observers.push(observer);
    }
    
    fn notify(&self, message: &str) {
        for observer in &self.observers {
            observer.update(message);
        }
    }
}

fn main() {
    let mut subject = Subject::new();
    
    subject.attach(Box::new(EmailObserver {
        email: "user@example.com".to_string(),
    }));
    subject.attach(Box::new(SmsObserver {
        phone: "123-456-7890".to_string(),
    }));
    
    subject.notify("New notification!");
}
```

### 构建者模式

```rust
#[derive(Debug)]
struct HttpRequest {
    url: String,
    method: String,
    headers: Vec<(String, String)>,
    body: Option<String>,
    timeout: u64,
}

impl HttpRequest {
    fn builder(url: &str) -> HttpRequestBuilder {
        HttpRequestBuilder::new(url)
    }
    
    fn send(&self) {
        println!("Sending {} request to {}", self.method, self.url);
        if let Some(body) = &self.body {
            println!("Body: {}", body);
        }
    }
}

struct HttpRequestBuilder {
    url: String,
    method: String,
    headers: Vec<(String, String)>,
    body: Option<String>,
    timeout: u64,
}

impl HttpRequestBuilder {
    fn new(url: &str) -> Self {
        HttpRequestBuilder {
            url: url.to_string(),
            method: "GET".to_string(),
            headers: Vec::new(),
            body: None,
            timeout: 30,
        }
    }
    
    fn method(mut self, method: &str) -> Self {
        self.method = method.to_string();
        self
    }
    
    fn header(mut self, key: &str, value: &str) -> Self {
        self.headers.push((key.to_string(), value.to_string()));
        self
    }
    
    fn body(mut self, body: &str) -> Self {
        self.body = Some(body.to_string());
        self
    }
    
    fn timeout(mut self, timeout: u64) -> Self {
        self.timeout = timeout;
        self
    }
    
    fn build(self) -> HttpRequest {
        HttpRequest {
            url: self.url,
            method: self.method,
            headers: self.headers,
            body: self.body,
            timeout: self.timeout,
        }
    }
}

fn main() {
    let request = HttpRequest::builder("https://api.example.com/users")
        .method("POST")
        .header("Content-Type", "application/json")
        .header("Authorization", "Bearer token123")
        .body(r#"{"name": "Alice"}"#)
        .timeout(60)
        .build();
    
    println!("{:?}", request);
    request.send();
}
```

## Trait 对象 vs 泛型

### Trait 对象（动态分发）

```rust
trait Draw {
    fn draw(&self);
}

struct Button {
    label: String,
}

impl Draw for Button {
    fn draw(&self) {
        println!("Drawing button: {}", self.label);
    }
}

struct TextField {
    placeholder: String,
}

impl Draw for TextField {
    fn draw(&self) {
        println!("Drawing text field: {}", self.placeholder);
    }
}

// 使用 trait 对象
fn draw_components(components: &[Box<dyn Draw>]) {
    for component in components {
        component.draw();
    }
}

fn main() {
    let components: Vec<Box<dyn Draw>> = vec![
        Box::new(Button { label: "Submit".to_string() }),
        Box::new(TextField { placeholder: "Enter name".to_string() }),
    ];
    
    draw_components(&components);
}
```

### 泛型（静态分发）

```rust
// 使用泛型
fn draw_component<T: Draw>(component: &T) {
    component.draw();
}

fn main() {
    let button = Button { label: "Submit".to_string() };
    let text_field = TextField { placeholder: "Enter name".to_string() };
    
    draw_component(&button);
    draw_component(&text_field);
}
```

### 性能对比

| 特性 | Trait 对象 | 泛型 |
|------|-----------|------|
| 分发方式 | 动态（运行时） | 静态（编译时） |
| 性能 | 稍慢（虚函数调用） | 快（内联优化） |
| 二进制大小 | 较小 | 较大（单态化） |
| 灵活性 | 高（运行时选择） | 低（编译时确定） |
| 使用场景 | 异构集合、插件 | 同构集合、性能关键 |

## 最佳实践

### 1. 优先使用组合

```rust
// ✅ 好：使用组合
struct User {
    profile: Profile,
    permissions: Permissions,
}

// ❌ 避免：过度使用继承模拟
// trait User: Profile + Permissions { }
```

### 2. 使用 trait 约束而不是具体类型

```rust
// ✅ 好：使用 trait 约束
fn process<T: Processor>(processor: &T) {
    // ...
}

// ❌ 避免：硬编码具体类型
fn process(processor: &ConcreteProcessor) {
    // ...
}
```

### 3. 合理使用封装

```rust
pub struct BankAccount {
    balance: f64,  // 私有字段
}

impl BankAccount {
    // 提供受控的访问方式
    pub fn deposit(&mut self, amount: f64) {
        if amount > 0.0 {
            self.balance += amount;
        }
    }
    
    pub fn withdraw(&mut self, amount: f64) -> Result<(), String> {
        if amount > self.balance {
            Err("Insufficient funds".to_string())
        } else {
            self.balance -= amount;
            Ok(())
        }
    }
    
    pub fn balance(&self) -> f64 {
        self.balance
    }
}
```
