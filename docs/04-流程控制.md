# Rust 教程 - 流程控制详解

## 本章概述

流程控制决定代码的执行路径。Rust 提供了丰富的流程控制结构，包括条件判断、循环和模式匹配。

## 学习目标

- 掌握 if 表达式的使用
- 理解三种循环结构
- 掌握 match 模式匹配
- 学会使用 if let 和 while let
- 理解模式匹配的高级用法

## 1. if 表达式

### 1.1 基本 if

```rust
let number = 7;

if number < 5 {
    println!("条件为真");
} else {
    println!("条件为假");
}
```

**重要**：条件必须是 `bool` 类型，不会自动转换。

### 1.2 else if

```rust
let number = 6;

if number % 4 == 0 {
    println!("number 可以被 4 整除");
} else if number % 3 == 0 {
    println!("number 可以被 3 整除");
} else if number % 2 == 0 {
    println!("number 可以被 2 整除");
} else {
    println!("number 不能被 4、3、2 整除");
}
```

### 1.3 if 作为表达式

`if` 是表达式，可以返回值：

```rust
let condition = true;
let number = if condition { 5 } else { 6 };

println!("结果: {}", number);
```

**注意**：两个分支必须返回相同类型。

```rust
// 错误示例
let number = if condition {
    5
} else {
    "six"  // 类型不匹配！
};
```

## 2. loop 循环

### 2.1 无限循环

`loop` 创建无限循环：

```rust
let mut counter = 0;

let result = loop {
    counter += 1;
    
    if counter == 10 {
        break counter * 2;  // break 可以返回值
    }
};

println!("结果: {}", result);  // 20
```

### 2.2 循环标签

处理嵌套循环时，使用标签指定要跳出哪一层：

```rust
let mut count = 0;

'counting_up: loop {
    println!("count = {}", count);
    let mut remaining = 10;
    
    loop {
        println!("  remaining = {}", remaining);
        if remaining == 9 {
            break;  // 跳出内层循环
        }
        if count == 2 {
            break 'counting_up;  // 跳出外层循环
        }
        remaining -= 1;
    }
    
    count += 1;
}
```

**标签语法**：使用单引号 `'` 开头。

## 3. while 循环

```rust
let mut number = 3;

while number != 0 {
    println!("倒计时: {}!", number);
    number -= 1;
}
println!("发射！");
```

**使用场景**：
- 不确定循环次数
- 基于条件循环

## 4. for 循环

### 4.1 遍历集合

```rust
let arr = [10, 20, 30, 40, 50];

for element in arr {
    println!("元素: {}", element);
}
```

### 4.2 Range

使用 Range 创建数字序列：

```rust
for number in 1..=5 {
    println!("Range 1..=5: {}", number);
}
```

**Range 类型**：
- `1..5`：包含1，不包含5
- `1..=5`：包含1和5

### 4.3 反向迭代

```rust
for number in (1..4).rev() {
    println!("反向 Range: {}", number);
}
```

### 4.4 for vs while

```rust
// 使用 for - 推荐
for element in &collection {
    println!("{}", element);
}

// 使用 while
let mut index = 0;
while index < collection.len() {
    println!("{}", collection[index]);
    index += 1;
}
```

**推荐使用 for**：
- 更简洁
- 更安全（不会越界）
- 更快

## 5. match 表达式

### 5.1 基本匹配

```rust
let number = 13;

match number {
    1 => println!("一"),
    2 | 3 | 5 | 7 | 11 => println!("质数"),
    13..=19 => println!("青少年数字"),
    _ => println!("其他数字"),
}
```

**注意**：必须穷尽所有可能，`_` 是通配符。

### 5.2 绑定值

```rust
match number {
    1 => println!("一"),
    n @ 2 | n @ 3 | n @ 5 => println!("质数 {}", n),
    n @ 13..=19 => println!("青少年数字 {}", n),
    n => println!("其他数字 {}", n),
}
```

### 5.3 匹配守卫

使用 `if` 添加额外条件：

```rust
let number = 4;

match number {
    n if n % 2 == 0 => println!("偶数: {}", n),
    n => println!("奇数: {}", n),
}
```

### 5.4 解构匹配

**元组解构**：

```rust
let point = (3, 5);

match point {
    (0, y) => println!("在 y 轴上，y = {}", y),
    (x, 0) => println!("在 x 轴上，x = {}", x),
    (x, y) => println!("坐标: ({}, {})", x, y),
}
```

**结构体解构**：

```rust
struct Point {
    x: i32,
    y: i32,
}

let point = Point { x: 10, y: 20 };

match point {
    Point { x: 0, y } => println!("在 y 轴上"),
    Point { x, y: 0 } => println!("在 x 轴上"),
    Point { x, y } => println!("坐标: ({}, {})", x, y),
}
```

**枚举解构**：

```rust
enum Message {
    Quit,
    Move { x: i32, y: i32 },
    Write(String),
}

match message {
    Message::Quit => println!("退出"),
    Message::Move { x, y } => println!("移动到 ({}, {})", x, y),
    Message::Write(text) => println!("写入: {}", text),
}
```

## 6. if let

### 6.1 基本用法

只关心一种模式时，比 `match` 更简洁：

```rust
let some_value = Some(7);

// 使用 match
match some_value {
    Some(7) => println!("匹配到 7"),
    _ => (),
}

// 使用 if let
if let Some(7) = some_value {
    println!("匹配到 7");
}
```

### 6.2 带 else

```rust
let some_value = Some(8);

if let Some(7) = some_value {
    println!("匹配到 7");
} else {
    println!("没匹配到 7");
}
```

### 6.3 链式 if let

```rust
if let Some(x) = option1 {
    // 处理 x
} else if let Some(y) = option2 {
    // 处理 y
} else {
    // 默认处理
}
```

## 7. while let

```rust
let mut optional = Some(0);

while let Some(i) = optional {
    if i > 5 {
        println!("Done!");
        optional = None;
    } else {
        println!("i = {}", i);
        optional = Some(i + 1);
    }
}
```

**使用场景**：迭代直到不符合条件。

## 8. let else

Rust 1.65 引入的语法：

```rust
let some_value: Option<i32> = Some(42);

let Some(value) = some_value else {
    println!("值不存在");
    return;  // 或 panic!, break 等
};

println!("值存在: {}", value);
```

**优势**：
- 早期返回模式更清晰
- 避免嵌套

## 9. 循环控制

### 9.1 break

立即退出循环：

```rust
for i in 1..=10 {
    if i == 3 {
        break;  // 退出循环
    }
    println!("{}", i);
}
```

### 9.2 continue

跳过当前迭代：

```rust
for i in 1..=5 {
    if i == 3 {
        continue;  // 跳过 3
    }
    println!("{}", i);
}
```

### 9.3 带值 break

```rust
let result = loop {
    let mut s = 0;
    for i in 1..=10 {
        s += i;
    }
    break s;
};
println!("结果: {}", result);
```

## 10. matches! 宏

判断值是否匹配模式：

```rust
let x = Some(5);

let is_match = matches!(x, Some(5));  // true
let is_even = matches!(x, Some(n) if n % 2 == 0);  // false
```

**使用场景**：
- 在 if 条件中使用
- 作为函数返回值

## 11. 最佳实践

### 选择合适的循环

```rust
// 已知迭代次数 - 使用 for
for i in 0..10 { }

// 基于条件 - 使用 while
while condition { }

// 无限循环或需要返回值 - 使用 loop
loop {
    if done {
        break result;
    }
}
```

### 使用 match 还是 if let

```rust
// 多个分支 - 使用 match
match value {
    Pattern1 => { },
    Pattern2 => { },
    Pattern3 => { },
    _ => { },
}

// 单个分支 - 使用 if let
if let Pattern = value {
    // 处理
}
```

### 避免深层嵌套

```rust
// 不好 - 深层嵌套
if condition1 {
    if condition2 {
        if condition3 {
            // 处理
        }
    }
}

// 好 - 早期返回
if !condition1 {
    return;
}
if !condition2 {
    return;
}
if !condition3 {
    return;
}
// 处理
```

## 常见问题

### Q1: for 和 while 哪个更快？

**答**：性能相近，for 更安全、更简洁。

### Q2: match 是否有性能开销？

**答**：编译器会优化 match，通常没有额外开销。

### Q3: if let 会检查所有模式吗？

**答**：不会，只检查指定的模式。
