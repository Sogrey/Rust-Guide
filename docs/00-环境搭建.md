# Rust 环境搭建

本教程将指导你如何在各个操作系统上搭建 Rust 开发环境。

## 安装 Rust

### Windows 系统

#### 方法一：使用 rustup-init.exe（推荐）

1. **下载安装程序**
   - 访问 [Rust 官网](https://www.rust-lang.org/zh-CN/tools/install)
   - 下载 `rustup-init.exe`

2. **运行安装程序**
   - 双击运行下载的文件
   - 选择安装选项（推荐选择默认选项 1）
   - 安装程序会自动下载并安装 Rust

3. **安装 C++ 构建工具**
   Rust 需要 Microsoft C++ 构建工具。如果尚未安装，安装程序会提示你安装：
   - 下载 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)
   - 安装时选择 "C++ 生成工具" 工作负载

4. **验证安装**
   ```powershell
   rustc --version
   cargo --version
   ```

#### 方法二：使用命令行（PowerShell）

```powershell
# 下载并运行安装脚本
Invoke-WebRequest -Uri https://win.rustup.rs/x86_64 -OutFile rustup-init.exe
.\rustup-init.exe
```

### Linux 系统

#### 一键安装

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

#### 手动安装步骤

1. **下载并运行安装脚本**
   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   ```

2. **配置环境变量**
   ```bash
   source $HOME/.cargo/env
   ```

3. **验证安装**
   ```bash
   rustc --version
   cargo --version
   ```

#### 安装依赖（某些 Linux 发行版）

Ubuntu/Debian:
```bash
sudo apt-get update
sudo apt-get install build-essential
```

Fedora/CentOS:
```bash
sudo dnf install gcc
```

### macOS 系统

#### 方法一：使用官方脚本（推荐）

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

#### 方法二：使用 Homebrew

```bash
brew install rust
```

> **注意**：使用 Homebrew 安装可能不是最新版本，推荐使用官方脚本。

#### 安装 Xcode 命令行工具

macOS 需要安装 Xcode 命令行工具：

```bash
xcode-select --install
```

## Rustup - Rust 版本管理器

`rustup` 是 Rust 的官方版本管理器，用于安装和管理 Rust 工具链。

### 常用命令

```bash
# 查看已安装的工具链
rustup show

# 更新 Rust 到最新版本
rustup update

# 安装特定版本
rustup install 1.75.0

# 设置默认工具链
rustup default stable

# 使用 nightly 版本
rustup install nightly
rustup default nightly

# 查看所有可用目标
rustup target list

# 添加交叉编译目标（例如添加 Linux 目标）
rustup target add x86_64-unknown-linux-gnu

# 查看帮助
rustup --help
```

### 工具链类型

- **stable**：稳定版本，推荐日常开发使用
- **beta**：测试版本，包含即将发布的功能
- **nightly**：每日构建版本，包含最新功能但可能不稳定

```bash
# 安装不同版本
rustup install stable
rustup install beta
rustup install nightly

# 在特定项目使用 nightly
cd my-project
rustup override set nightly
```

## 配置镜像源（中国大陆用户）

如果在中国大陆，下载速度可能较慢，可以配置国内镜像源。

### 配置环境变量（临时）

在 PowerShell（Windows）或终端（Linux/macOS）中设置：

```bash
# Windows PowerShell
$env:RUSTUP_DIST_SERVER = "https://rsproxy.cn"
$env:RUSTUP_UPDATE_ROOT = "https://rsproxy.cn/rustup"

# Linux/macOS
export RUSTUP_DIST_SERVER="https://rsproxy.cn"
export RUSTUP_UPDATE_ROOT="https://rsproxy.cn/rustup"
```

### 永久配置

#### Windows

1. 右键"此电脑" → "属性" → "高级系统设置"
2. 点击"环境变量"
3. 在"用户变量"中新建：
   - 变量名：`RUSTUP_DIST_SERVER`
   - 变量值：`https://rsproxy.cn`
   - 变量名：`RUSTUP_UPDATE_ROOT`
   - 变量值：`https://rsproxy.cn/rustup`

或使用 PowerShell：

```powershell
[Environment]::SetEnvironmentVariable("RUSTUP_DIST_SERVER", "https://rsproxy.cn", "User")
[Environment]::SetEnvironmentVariable("RUSTUP_UPDATE_ROOT", "https://rsproxy.cn/rustup", "User")
```

#### Linux/macOS

编辑 `~/.bashrc` 或 `~/.zshrc`：

```bash
echo 'export RUSTUP_DIST_SERVER="https://rsproxy.cn"' >> ~/.bashrc
echo 'export RUSTUP_UPDATE_ROOT="https://rsproxy.cn/rustup"' >> ~/.bashrc
source ~/.bashrc
```

### 配置 Cargo 镜像

编辑 `~/.cargo/config`（Linux/macOS）或 `%USERPROFILE%\.cargo\config`（Windows）：

```toml
[source.crates-io]
replace-with = 'rsproxy-sparse'

[source.rsproxy]
registry = "https://rsproxy.cn/crates.io-index"

[source.rsproxy-sparse]
registry = "sparse+https://rsproxy.cn/index/"

[registries.rsproxy]
index = "https://rsproxy.cn/crates.io-index"

[net]
git-fetch-with-cli = true
```

### 其他镜像源

- **中科大镜像**：`https://mirrors.ustc.edu.cn/rust-static`
- **清华大学镜像**：`https://mirrors.tuna.tsinghua.edu.cn/rustup`

## IDE 和编辑器配置

### Visual Studio Code（推荐）

#### 安装插件

1. **rust-analyzer**（必装）
   - 官方 Rust 语言服务器
   - 提供代码补全、跳转、重构等功能
   - 性能优秀，功能强大

2. **CodeLLDB**（推荐）
   - 调试器支持
   - 支持断点、变量查看等功能

3. **Better TOML**（推荐）
   - TOML 文件语法高亮
   - Cargo.toml 文件编辑支持

4. **Error Lens**（推荐）
   - 行内显示错误信息
   - 提高开发效率

5. ** crates**（可选）
   - 在 Cargo.toml 中显示 crate 版本
   - 帮助管理依赖版本

#### 配置文件

在项目根目录创建 `.vscode/settings.json`：

```json
{
  "rust-analyzer.cargo.features": "all",
  "rust-analyzer.checkOnSave.command": "clippy",
  "rust-analyzer.procMacro.enable": true,
  "editor.formatOnSave": true,
  "[rust]": {
    "editor.defaultFormatter": "rust-lang.rust-analyzer"
  }
}
```

#### 调试配置

创建 `.vscode/launch.json`：

```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "lldb",
      "request": "launch",
      "name": "Debug",
      "cargo": {
        "args": ["build", "--bin=${fileBasenameNoExtension}", "--package=${workspaceFolderBasename}"],
        "filter": {
          "name": "${fileBasenameNoExtension}",
          "kind": "bin"
        }
      },
      "args": [],
      "cwd": "${workspaceFolder}"
    }
  ]
}
```

## 验证安装

### 创建测试项目

```bash
# 创建新项目
cargo new hello_rust

# 进入项目目录
cd hello_rust

# 运行项目
cargo run
```

如果看到输出 `Hello, world!`，说明安装成功！

### 测试编译器

创建 `test.rs` 文件：

```rust
fn main() {
    println!("Rust 版本: {}", env!("RUSTC_VERSION"));
    println!("操作系统: {}", std::env::consts::OS);
    println!("架构: {}", std::env::consts::ARCH);
}
```

编译运行：

```bash
rustc test.rs
./test  # Linux/macOS
test.exe  # Windows
```

## 工具组件

Rust 生态系统提供了许多有用的工具：

### Cargo - 包管理器

```bash
# 创建项目
cargo new project_name

# 构建项目
cargo build

# 运行项目
cargo run

# 运行测试
cargo test

# 生成文档
cargo doc

# 检查代码（不生成二进制）
cargo check
```

### Rustfmt - 代码格式化

```bash
# 格式化当前目录所有 Rust 文件
cargo fmt

# 格式化特定文件
rustfmt src/main.rs

# 检查格式但不修改
cargo fmt -- --check
```

### Clippy - 代码检查

```bash
# 运行 clippy 检查
cargo clippy

# 自动修复部分问题
cargo clippy --fix
```

### Rustdoc - 文档生成

```bash
# 生成项目文档
cargo doc

# 生成并打开文档
cargo doc --open
```

## 常见问题

### 1. 安装后提示 "cargo 不是内部或外部命令"

**解决方案**：
- 重启终端或命令提示符
- Windows：检查环境变量 PATH 中是否包含 `%USERPROFILE%\.cargo\bin`
- Linux/macOS：运行 `source $HOME/.cargo/env`

### 2. 链接器错误

**错误信息**：`linker 'cc' not found`

**解决方案**：
- Linux：安装 `build-essential`（Ubuntu/Debian）或 `gcc`（Fedora/CentOS）
- Windows：安装 Visual Studio Build Tools
- macOS：安装 Xcode 命令行工具

### 3. 更新 Rust 后编译错误

**解决方案**：
```bash
# 更新工具链
rustup update

# 如果使用 nightly，可能需要回退到稳定版
rustup default stable

# 清理并重新构建
cargo clean
cargo build
```

### 4. Windows 上编译慢

**解决方案**：
- 使用更快的链接器：在 `.cargo/config` 中添加：
  ```toml
  [target.x86_64-pc-windows-msvc]
  linker = "rust-lld"
  ```
- 或使用 GNU 工具链：`rustup default stable-x86_64-pc-windows-gnu`
