# Rust 教程 - 数据类型详解

## 本章概述

Rust 是静态类型语言，编译时必须知道所有变量的类型。本章详细介绍 Rust 的数据类型系统，包括标量类型和复合类型。

## 学习目标

- 掌握各种标量类型
- 理解复合类型
- 学会类型转换
- 理解类型推断
- 掌握类型别名

## 1. 标量类型

标量类型表示单个值，Rust 有四种基本标量类型。

### 1.1 整数类型

Rust 提供了多种整数类型：

| 长度 | 有符号 | 无符号 |
|------|--------|--------|
| 8-bit | i8 | u8 |
| 16-bit | i16 | u16 |
| 32-bit | i32 | u32 |
| 64-bit | i64 | u64 |
| 128-bit | i128 | u128 |
| arch | isize | usize |

**有符号 vs 无符号**：
- 有符号：可存储正负数，范围 `-2^(n-1)` 到 `2^(n-1)-1`
- 无符号：只能存储非负数，范围 `0` 到 `2^n-1`

```rust
let i8_val: i8 = -128;        // -128 到 127
let u8_val: u8 = 255;         // 0 到 255
let i32_val: i32 = -100;      // 默认整数类型
```

**整数字面量表示法**：

```rust
let decimal = 98_222;         // 十进制，下划线提高可读性
let hex = 0xff;               // 十六进制
let octal = 0o77;             // 八进制
let binary = 0b1111_0000;     // 二进制
let byte = b'A';              // 字节（仅限 u8）
```

**整数溢出**：
- Debug 模式：溢出会 panic
- Release 模式：溢出会回绕（wrapping）

### 1.2 浮点类型

Rust 有两种浮点类型：

```rust
let x = 2.0;      // f64，默认浮点类型（双精度）
let y: f32 = 3.0; // f32，单精度
```

**推荐使用 f64**：
- 现代CPU上速度与f32相当
- 精度更高
- Rust默认类型

**浮点运算**：

```rust
let sum = 2.0 + 5.0;
let product = 2.0 * 3.5;
let quotient = 10.0 / 3.0;
```

**注意**：浮点数遵循 IEEE-754 标准，存在精度问题。

### 1.3 布尔类型

布尔类型只有两个值：`true` 和 `false`。

```rust
let t = true;
let f: bool = false;  // 显式类型注解
```

**主要用途**：
- 条件判断
- 标志位
- 逻辑运算

### 1.4 字符类型

`char` 类型是 Rust 最原生的字母类型：

```rust
let c = 'z';
let z: char = 'ℤ';
let heart_eyed_cat = '😻';
let chinese = '中';
let emoji = '🦀';
```

**重要特性**：
- 大小：4字节
- 表示：Unicode 标量值
- 范围：U+0000 到 U+D7FF 和 U+E000 到 U+10FFFF

## 2. 复合类型

复合类型可以将多个值组合成一个类型。

### 2.1 元组（Tuple）

元组可以将不同类型的值组合在一起：

```rust
let tup: (i32, f64, u8) = (500, 6.4, 1);
println!("元组: {:?}", tup);
```

**解构元组**：

```rust
let (x, y, z) = tup;
println!("解构后: x = {}, y = {}, z = {}", x, y, z);
```

**通过索引访问**：

```rust
let five_hundred = tup.0;
let six_point_four = tup.1;
let one = tup.2;
```

**单元素元组**：

```rust
let single = (5,);  // 注意逗号
let not_a_tuple = (5);  // 这只是一个括号内的整数！
```

### 2.2 数组（Array）

数组是固定长度、类型相同的元素集合，存储在栈上：

```rust
let arr = [1, 2, 3, 4, 5];
println!("数组: {:?}", arr);
```

**指定类型和长度**：

```rust
let arr2: [i32; 5] = [1, 2, 3, 4, 5];
```

**重复初始化**：

```rust
let arr3 = [3; 5];  // 等同于 [3, 3, 3, 3, 3]
```

**访问元素**：

```rust
let first = arr[0];
let second = arr[1];
```

**数组切片**：

```rust
let slice = &arr[1..3];  // 获取索引 1 和 2 的元素
```

**数组 vs Vector**：
- 数组：固定长度，栈上分配
- Vector：动态长度，堆上分配

## 3. 类型推断

Rust 编译器会自动推断类型：

```rust
let inferred = 10;       // 推断为 i32
let inferred_float = 3.0; // 推断为 f64
```

**根据使用推断**：

```rust
let mut vec = Vec::new();
vec.push(1);  // 现在编译器知道是 Vec<i32>
```

## 4. 类型别名

使用 `type` 关键字创建类型别名：

```rust
type Kilometers = i32;

let distance: Kilometers = 100;
println!("距离: {} 公里", distance);
```

**使用场景**：
- 简化复杂类型
- 提高代码可读性
- 方便类型修改

## 5. 类型转换

使用 `as` 关键字进行显式转换：

```rust
let decimal = 65.4321;
let integer = decimal as i32;  // 截断小数部分
println!("浮点转整数: {} -> {}", decimal, integer);
```

**常见转换**：

```rust
let integer = 65u8;
let character = integer as char;  // u8 可以直接转换为 char

// 使用 char::from_u32
let unicode_val: u32 = 0x1F600;  // 😀 emoji
if let Some(emoji) = char::from_u32(unicode_val) {
    println!("Unicode 转 char: {} -> {}", unicode_val, emoji);
}
```

**注意溢出**：

```rust
let large: i32 = 300;
let small: i8 = large as i8;  // 溢出！结果是 44
```

## 6. 空类型

### 单元类型

单元类型 `()` 表示没有有意义的值：

```rust
let unit = ();
println!("unit 类型: {:?}", unit);
```

**用途**：
- 函数没有返回值
- 不关心返回值

### never 类型

`!` 类型表示永远不返回：

```rust
fn always_panic() -> ! {
    panic!("永远不返回");
}
```

**用途**：
- `panic!` 宏
- `continue` 和 `break`
- 死循环

## 7. 常用操作

### 类型检查

```rust
fn type_of<T>(_: &T) -> &str {
    std::any::type_name::<T>()
}

let x = 42;
println!("x 的类型: {}", type_of(&x));  // i32
```

### 类型比较

```rust
use std::any::TypeId;

fn is_same_type<T: 'static, U: 'static>(_: T, _: U) -> bool {
    TypeId::of::<T>() == TypeId::of::<U>()
}
```

## 8. 最佳实践

### 选择合适的整数类型

```rust
// 小范围：使用较小的类型节省内存
let age: u8 = 25;  // 年龄不会超过 255

// 一般用途：使用默认的 i32
let count = 100;

// 大数：使用 i64 或 i128
let big_number: i64 = 1_000_000_000_000;
```

### 数组 vs Vector

```rust
// 已知固定大小且较小 - 使用数组
let days_in_week = ["Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun"];

// 动态大小或较大 - 使用 Vector
let mut numbers = Vec::new();
numbers.push(1);
```

### 避免类型转换

```rust
// 不好 - 频繁转换
let x: i32 = 10;
let y: i64 = x as i64 + 100;

// 好 - 使用合适的类型
let x: i64 = 10;
let y = x + 100;
```

## 常见问题

### Q1: i32 和 i64 该选哪个？

**答**：一般使用 `i32`（Rust 默认），除非：
- 需要更大范围，使用 `i64`
- 需要优化内存，使用较小类型

### Q2: 为什么数组的越界检查在运行时而不是编译时？

**答**：Rust 在编译时检查常量索引，但运行时检查变量索引，这是因为编译时无法知道变量的值。

### Q3: 字符和字符串有什么区别？

**答**：
- `char`：4字节，单个Unicode字符
- `String`：堆分配，UTF-8编码，可变长度字符串
