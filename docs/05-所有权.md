# Rust 教程 - 所有权详解

## 本章概述

所有权是 Rust 最独特的特性，它让 Rust 无需垃圾回收器就能保证内存安全。理解所有权是掌握 Rust 的关键。

## 学习目标

- 理解所有权三大规则
- 掌握移动语义
- 理解克隆和复制
- 掌握引用和借用
- 理解切片类型
- 掌握所有权实战应用

## 1. 所有权规则

Rust 的核心所有权规则：

1. **每个值都有一个所有者**（owner）
2. **同一时刻只能有一个所有者**
3. **当所有者离开作用域，值将被丢弃**

```rust
{
    let s = String::from("hello");  // s 从这里开始有效
    
    // 使用 s
}  // s 离开作用域，被丢弃
```

## 2. 变量作用域

作用域是变量有效的范围：

```rust
{                      // s 在这里无效，尚未声明
    let s = "hello";   // s 从这里开始有效
    
    // 使用 s
}                      // s 离开作用域，无效
```

## 3. String 类型

理解所有权需要先理解堆分配的数据类型。

### 3.1 字符串字面量 vs String

```rust
// 字符串字面量 - 不可变，存储在二进制中
let s1: &str = "hello";

// String - 可变，存储在堆上
let mut s2 = String::from("hello");
s2.push_str(", world!");  // 可以修改
```

### 3.2 String 的内存布局

String 由三部分组成：
- **ptr**：指向堆上的数据
- **len**：当前长度
- **capacity**：总容量

```
栈上：ptr | len | capacity
        ↓
堆上：h | e | l | l | o
```

## 4. 移动语义

### 4.1 变量赋值

```rust
let s1 = String::from("hello");
let s2 = s1;  // s1 的所有权移动到 s2

// println!("{}", s1);  // 错误！s1 已失效
println!("{}", s2);     // 正常
```

### 4.2 为什么需要移动？

**问题**：如果只复制指针：
- 两个指针指向同一数据
- 释放时会 double free
- 复制所有数据会很慢

**解决方案**：移动所有权，使原变量失效。

### 4.3 函数传值

```rust
fn takes_ownership(some_string: String) {
    println!("{}", some_string);
}  // some_string 离开作用域，被丢弃

let s = String::from("hello");
takes_ownership(s);

// println!("{}", s);  // 错误！s 已失效
```

### 4.4 函数返回值

```rust
fn gives_ownership() -> String {
    String::from("hello")
}

let s = gives_ownership();  // 所有权移动给 s
```

## 5. 克隆（Clone）

深拷贝，显式复制：

```rust
let s1 = String::from("hello");
let s2 = s1.clone();  // 深拷贝

println!("s1 = {}, s2 = {}", s1, s2);  // 两个都有效
```

**何时使用**：
- 确实需要两份数据
- 数据量不大

## 6. 复制（Copy）

栈上数据的特殊行为：

```rust
let x = 5;
let y = x;  // 复制，不是移动

println!("x = {}, y = {}", x, y);  // 两个都有效
```

**实现 Copy 的类型**：
- 所有整数类型：u8, i32, usize 等
- 浮点类型：f32, f64
- 布尔类型：bool
- 字符类型：char
- 元组（所有元素都是 Copy）
- 不可变引用 &T

**不能实现 Copy**：
- `String`：需要释放堆内存
- `Vec<T>`：需要释放堆内存
- 可变引用 `&mut T`

## 7. 引用与借用

### 7.1 引用

引用允许使用值但不获取所有权：

```rust
fn calculate_length(s: &String) -> usize {
    s.len()
}  // s 离开作用域，但它只是引用，不会丢弃数据

let s1 = String::from("hello");
let len = calculate_length(&s1);  // 传递引用

println!("'{}' 的长度是 {}", s1, len);  // s1 仍然有效
```

**引用规则**：
- 默认不可变
- 引用总是有效的（不会悬垂）

### 7.2 可变引用

```rust
fn change(s: &mut String) {
    s.push_str(", world");
}

let mut s = String::from("hello");
change(&mut s);

println!("{}", s);  // hello, world
```

### 7.3 引用规则

**规则一**：可以同时有多个不可变引用，或者一个可变引用

```rust
let mut s = String::from("hello");

// 正确 - 多个不可变引用
let r1 = &s;
let r2 = &s;
println!("{} {}", r1, r2);

// 正确 - 一个可变引用
let r3 = &mut s;
r3.push_str(" world");
```

**规则二**：引用的作用域

```rust
let mut s = String::from("hello");

let r1 = &s;  // 正确
let r2 = &s;  // 正确
println!("{} {}", r1, r2);
// r1 和 r2 最后一次使用在这里

let r3 = &mut s;  // 正确
r3.push_str(" world");
```

### 7.4 为什么有这些规则？

**数据竞争**：三个条件同时满足：
1. 两个或多个指针同时访问同一数据
2. 至少有一个指针在写入
3. 没有同步机制

Rust 在编译时防止数据竞争！

## 8. 悬垂引用

Rust 保证不会出现悬垂引用：

```rust
fn no_dangle() -> String {
    let s = String::from("hello");
    s  // 返回所有权，而不是引用
}

// 错误示例
fn dangle() -> &String {  // 错误！
    let s = String::from("hello");
    &s  // 返回局部变量的引用
}  // s 被丢弃，引用失效
```

## 9. 切片（Slice）

切片是引用部分数据。

### 9.1 字符串切片

```rust
let s = String::from("hello world");

let hello = &s[0..5];   // 或 &s[..5]
let world = &s[6..11];  // 或 &s[6..]
let full = &s[..];      // 整个字符串
```

**注意**：切片索引必须是有效的 UTF-8 字符边界。

### 9.2 字符串切片类型

```rust
let s = String::from("hello");  // String 类型
let slice: &str = &s[0..2];     // &str 类型
```

### 9.3 其他切片

```rust
let arr = [1, 2, 3, 4, 5];
let slice: &[i32] = &arr[1..3];
```

## 10. 实战示例

### 10.1 查找第一个单词

```rust
fn first_word(s: &str) -> &str {
    let bytes = s.as_bytes();
    
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }
    
    &s[..]
}

let s = String::from("hello world");
let word = first_word(&s);  // "hello"
```

### 10.2 较长的字符串

```rust
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}
```

## 11. 所有权总结

| 操作 | 栈数据（Copy） | 堆数据（如 String） |
|------|----------------|---------------------|
| 赋值 | 复制 | 移动 |
| 函数传参 | 复制 | 移动 |
| 函数返回 | 复制 | 移动 |
| 引用 | 引用 | 引用 |

## 12. 最佳实践

### 优先使用引用

```rust
// 好 - 使用引用
fn process(data: &String) {
    println!("{}", data);
}

// 避免 - 获取所有权
fn process(data: String) {
    println!("{}", data);
}
```

### 返回所有权而非引用

```rust
// 好 - 返回所有权
fn create_string() -> String {
    String::from("hello")
}

// 避免 - 返回局部引用
fn create_string() -> &String {
    // 编译错误！
}
```

### 使用切片参数

```rust
// 好 - 接受字符串切片
fn first_word(s: &str) -> &str {
    // ...
}

// 可以接受 &String 和 &str
let s = String::from("hello");
first_word(&s);      // &String 自动转换为 &str
first_word("hello"); // 直接传递 &str
```

## 常见问题

### Q1: 为什么 Rust 需要所有权？

**答**：
- 无需垃圾回收器
- 保证内存安全
- 零运行时开销

### Q2: 如何选择移动还是克隆？

**答**：
- 性能敏感：使用引用或移动
- 需要独立副本：使用克隆

### Q3: 为什么可变引用只能有一个？

**答**：防止数据竞争，保证内存安全。
