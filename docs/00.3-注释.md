# Rust 注释

注释是代码中的说明文字，不会被编译器编译。Rust 支持三种注释方式。

## 普通注释

### 行注释

使用 `//` 注释单行：

```rust
fn main() {
    // 这是一个行注释
    // 编译器会忽略注释内容
    
    let x = 5; // 行尾注释
    
    // 注释可以用来临时禁用代码
    // let y = 10;
    
    // 注释也可以很长很长很长很长很长很长很长很长很长很长
    // 长到一行写不下时，换行继续
    
    // 注释用于解释复杂的逻辑
    // 计算斐波那契数列的第 n 项
    // 使用递归方法，时间复杂度 O(2^n)
    let fib_10 = fibonacci(10);
    println!("{}", fib_10);
}

fn fibonacci(n: u32) -> u32 {
    if n <= 1 {
        n
    } else {
        fibonacci(n - 1) + fibonacci(n - 2)
    }
}
```

### 块注释

使用 `/* ... */` 注释多行：

```rust
fn main() {
    /* 这是一个块注释
       可以跨越多行
       直到遇到结束标记 */
    
    /*
     * 常见的块注释风格
     * 每行开头加星号
     * 看起来更整齐
     */
    
    let x = /* 也可以在代码中间使用块注释 */ 5;
    
    /*
    块注释可以临时禁用大段代码
    
    fn old_function() {
        println!("This is old code");
        let x = 10;
        let y = 20;
    }
    */
}
```

### 块注释嵌套

Rust 的块注释可以嵌套（这是一个很棒的特性）：

```rust
fn main() {
    /* 外层注释开始
       /* 内层注释 */
       外层注释继续 */
    
    let outer = 5;
    /* 注释 A
       /* 注释 B
          /* 注释 C */
          B 继续 */
       A 继续 */
    let inner = 10;
}
```

这对临时注释掉包含块注释的代码非常有用：

```rust
fn main() {
    /*
    let x = 5;
    /* 这段代码已经有块注释了 */
    let y = 10;
    */
}
```

## 文档注释

文档注释是 Rust 的重要特性，用于生成 API 文档。

### 外层文档注释

使用 `///` 或 `//!`：

- `///` - 用于紧随其后的项（函数、结构体等）
- `//!` - 用于包含它的项（通常是模块或 crate）

```rust
/// 这是外层文档注释
/// 用于说明紧随其后的函数
/// 
/// # Examples
/// 
/// ```
/// let result = add(2, 3);
/// assert_eq!(result, 5);
/// ```
fn add(a: i32, b: i32) -> i32 {
    a + b
}

/// 计算两个数的乘积
/// 
/// # Arguments
/// 
/// * `a` - 第一个数
/// * `b` - 第二个数
/// 
/// # Returns
/// 
/// 返回 `a` 和 `b` 的乘积
/// 
/// # Panics
/// 
/// 如果结果溢出会 panic
fn multiply(a: i32, b: i32) -> i32 {
    a * b
}

/// 一个表示点的结构体
/// 
/// # Examples
/// 
/// ```
/// let p = Point::new(10, 20);
/// assert_eq!(p.x, 10);
/// assert_eq!(p.y, 20);
/// ```
#[derive(Debug)]
struct Point {
    /// x 坐标
    x: i32,
    /// y 坐标
    y: i32,
}

impl Point {
    /// 创建新的 Point 实例
    /// 
    /// # Arguments
    /// 
    /// * `x` - x 坐标
    /// * `y` - y 坐标
    /// 
    /// # Examples
    /// 
    /// ```
    /// let p = Point::new(5, 10);
    /// ```
    fn new(x: i32, y: i32) -> Self {
        Point { x, y }
    }
}
```

### 内层文档注释

使用 `//!` 注释模块或 crate：

```rust
//! # Rust 学习示例
//! 
//! 这个 crate 提供了 Rust 学习的示例代码
//! 
//! ## 模块
//! 
//! - `math` - 数学运算
//! - `utils` - 工具函数

//! 这个模块提供数学运算功能
//! 
//! 包含加减乘除等基本运算

pub mod math {
    //! 数学运算模块
    
    /// 两数相加
    pub fn add(a: i32, b: i32) -> i32 {
        a + b
    }
}
```

### 文档注释格式

#### Markdown 支持

文档注释支持 Markdown 语法：

```rust
/// # 大标题
/// 
/// ## 中标题
/// 
/// ### 小标题
/// 
/// 普通文本，**粗体**，*斜体*，`代码`
/// 
/// - 列表项 1
/// - 列表项 2
/// 
/// 1. 有序列表 1
/// 2. 有序列表 2
/// 
/// > 引用块
/// 
/// ```
/// // 代码块
/// let x = 5;
/// ```
/// 
/// [链接](https://www.rust-lang.org/)
fn example() {}
```

#### 常用章节

文档注释中有一些约定俗成的章节：

```rust
/// 用户信息结构体
/// 
/// # Examples
/// 
/// 基本使用：
/// 
/// ```
/// let user = User::new("Alice".to_string(), 30);
/// println!("{}", user.name);
/// ```
/// 
/// # Arguments
/// 
/// * `name` - 用户名
/// * `age` - 年龄（必须 >= 0）
/// 
/// # Returns
/// 
/// 返回一个新的 User 实例
/// 
/// # Panics
/// 
/// 如果 age < 0 会 panic
/// 
/// # Errors
/// 
/// 如果创建失败会返回 `UserError`
/// 
/// # Safety
/// 
/// 这个函数是安全的，不需要 unsafe 块
/// 
/// # Notes
/// 
/// 注意年龄应该合理
struct User {
    name: String,
    age: u32,
}

impl User {
    fn new(name: String, age: u32) -> Self {
        User { name, age }
    }
}
```

#### 代码块标注

代码块可以标注类型：

```rust
/// # Examples
/// 
/// 普通代码块（会编译测试）：
/// 
/// ```
/// let x = 5;
/// ```
/// 
/// 忽略编译（不测试）：
/// 
/// ```ignore
/// let x = non_existent_function();
/// ```
/// 
/// 编译失败测试：
/// 
/// ```compile_fail
/// let x: i32 = "hello";  // 类型错误
/// ```
/// 
/// 不运行测试：
/// 
/// ```no_run
/// loop {
///     // 无限循环，不运行
/// }
/// ```
fn example() {}
```

## 生成文档

### 使用 cargo doc

```bash
# 生成项目文档
cargo doc

# 生成并打开文档
cargo doc --open

# 包含依赖的文档
cargo doc --document-private-items

# 只生成当前 crate 的文档
cargo doc --no-deps
```

### 文档属性

使用属性控制文档生成：

```rust
/// 这个函数有文档
pub fn documented_function() {}

/// 这个函数没有文档
#[doc(hidden)]
pub fn undocumented_function() {}

/// 这个函数的文档被标记为隐藏
/// 但仍然可以通过特殊方式访问
pub fn semi_hidden_function() {}

// 忽略文档测试
#[doc = "这个函数有简单的文档"]
#[doc(alias = "short_name")]  // 文档别名
pub fn long_function_name() {}

// 使用 rustdoc 属性
#[cfg_attr(doctest, doc = "```ignore")]
fn special_example() {}
```

### 包含外部文件

```rust
// 从外部文件包含文档
#[doc = include_str!("../README.md")]
pub mod my_module {
    //! 模块文档
}
```

## 注释最佳实践

### 1. 解释 "为什么" 而不是 "是什么"

```rust
// ❌ 不好的注释
let x = x + 1;  // x 加 1

// ✅ 好的注释
let x = x + 1;  // 补偿 off-by-one 错误

// ❌ 不好的注释
// 遍历数组
for item in items {
    println!("{}", item);
}

// ✅ 好的注释
// 打印所有订单，用于调试日志
for item in items {
    println!("{}", item);
}
```

### 2. 保持注释更新

```rust
// ❌ 过时的注释
// 计算两个数的和
fn add(a: i32, b: i32) -> i32 {
    a * b  // 实际是乘法！
}

// ✅ 更新注释
// 计算两个数的乘积
fn multiply(a: i32, b: i32) -> i32 {
    a * b
}
```

### 3. 使用文档注释而不是普通注释

```rust
// ❌ 使用普通注释
// 用户信息结构体
pub struct User {
    // 用户名
    pub name: String,
}

// ✅ 使用文档注释
/// 用户信息结构体
pub struct User {
    /// 用户名
    pub name: String,
}
```

### 4. 包含示例代码

```rust
/// 计算阶乘
/// 
/// # Examples
/// 
/// ```
/// use mylib::factorial;
/// 
/// assert_eq!(factorial(5), 120);
/// assert_eq!(factorial(0), 1);
/// ```
pub fn factorial(n: u32) -> u32 {
    if n <= 1 {
        1
    } else {
        n * factorial(n - 1)
    }
}
```

### 5. 使用 TODO 和 FIXME

```rust
fn process_data(data: &str) -> Result<(), Error> {
    // TODO: 添加数据验证
    // FIXME: 处理空字符串的情况
    
    // HACK: 临时解决方案，需要重构
    let _ = data;
    Ok(())
}
```

### 6. 避免注释掉的代码

```rust
// ❌ 不要这样
// let old_value = 10;
// let new_value = old_value + 5;
// println!("{}", new_value);

// ✅ 使用版本控制
let new_value = 15;
println!("{}", new_value);
```

## 文档测试

文档中的代码块会被自动测试：

```rust
/// 将字符串转换为大写
/// 
/// # Examples
/// 
/// ```
/// use mylib::to_upper;
/// 
/// let result = to_upper("hello");
/// assert_eq!(result, "HELLO");
/// ```
pub fn to_upper(s: &str) -> String {
    s.to_uppercase()
}
```

运行测试：

```bash
# 运行所有测试，包括文档测试
cargo test

# 只运行文档测试
cargo test --doc
```

## 实用技巧

### 使用注释调试

```rust
fn main() {
    let a = 1;
    let b = 2;
    // let c = a + b;  // 临时禁用
    let c = a * b;     // 测试新逻辑
    println!("{}", c);
}
```

### 条件编译注释

```rust
// 仅在调试模式编译
#[cfg(debug_assertions)]
fn debug_only() {
    println!("Debug mode");
}

// 通过配置开关
#[cfg(feature = "advanced")]
fn advanced_feature() {
    println!("Advanced feature");
}
```

### 自动生成文档模板

在 VS Code 等编辑器中，输入 `///` 会自动生成文档模板：

```rust
/// [summary]
/// 
/// # Arguments
/// 
/// * `arg1` - [description]
/// 
/// # Returns
/// 
/// [description]
fn my_function(arg1: i32) -> i32 {
    arg1
}
```

## 注释类型总结

| 类型 | 符号 | 用途 | 是否生成文档 |
|------|------|------|-------------|
| 行注释 | `//` | 代码说明、临时禁用 | 否 |
| 块注释 | `/* */` | 多行说明、临时禁用大段代码 | 否 |
| 外层文档注释 | `///` | 说明紧随其后的项 | 是 |
| 内层文档注释 | `//!` | 说明模块或 crate | 是 |

## 常见问题

### 1. 文档测试失败

**错误**：文档中的代码块编译失败

**解决方案**：
- 使用 `ignore` 标记不需要测试的代码块
- 使用 `no_run` 标记会无限运行的代码
- 使用 `compile_fail` 测试应该失败的代码

```rust
/// ```
/// // 这个代码块会被忽略
/// ```
/// ```ignore
/// let x = unknown_function();
/// ```
```

### 2. 中文注释乱码

**解决方案**：确保源文件使用 UTF-8 编码

```rust
// 大多数现代编辑器默认使用 UTF-8
// 如果遇到乱码，检查文件编码
fn 中文函数名() {  // Rust 支持非 ASCII 标识符
    println!("Hello");
}
```

### 3. 注释格式化

**解决方案**：使用 `rustfmt` 自动格式化注释

```bash
cargo fmt
```
